# OCR Quality Iteration Design

## 목적

- 현재 `AI-Repository-fresh`의 OCR 파이프라인을 기준으로 품질 개선 작업을 반복 가능하게 만든다.
- 감으로 규칙을 추가하지 않고, 실패 샘플과 회귀 테스트를 함께 관리한다.
- 목표는 `속도는 유지하면서`, `편의점/POS 실사 영수증의 false positive를 줄이는 것`이다.

## 현재 상태 요약

- 쉬운 샘플(`img2.items-crop.jpg`)은 품목 2개와 날짜를 안정적으로 뽑는다.
- 어려운 샘플(`img3.items-crop.jpg`, `SE-...items-crop.jpg`)은 다음 문제가 남아 있다.
  - 쿠폰/매출코드/봉투/세금 줄이 품목으로 섞임
  - 상품명 줄의 `5입`, `1` 같은 숫자를 금액으로 오인함
  - `상품명 + 다음 줄 숫자` 패턴과 `바코드 + 수량 + 금액` 패턴이 섞일 때 줄 조립이 흔들림

## 개선 전략

### 1. 문서화

- 품질 기준 문서를 별도로 둔다.
- 문서에는 샘플별 결과, 현재 한계, 다음 우선순위를 기록한다.

### 2. 회귀 테스트 우선

- `img2` 스타일
  - 품목 2개만 추출
  - 보증금/물품가액 줄 제외
- `img3` 스타일
  - 쿠폰/매출코드/봉투 줄 제외
  - `상품명 + 다음 숫자 줄` 패턴 허용
  - `(5입)`의 `5`를 가격으로 오인하지 않음

### 3. 파서 규칙 고도화

- 구조 노이즈 키워드 강화
  - `쿠폰`, `코드`, `봉투`, `재사용`, `물품가액`, `부가세`, `결제`
- generic single-line fallback을 더 보수적으로 운영
  - 콤마/원 표시가 없는 숫자는 가격 후보에서 제외
- `상품명 + 다음 숫자 줄` 패턴 추가
  - 예: `롯데 앤디카페조릿 다크` 다음 줄 `4,800 1 4.800`

### 4. 검증 기준

- 테스트 100% 통과
- `img2.items-crop.jpg`
  - 품목 수 2
- `img3.items-crop.jpg`
  - 품목 수가 기존 9개 수준에서 의미 있게 감소
  - 쿠폰/매출코드/봉투 줄 제외
- API warm 응답 시간은 1~2초대 유지

## 이번 라운드의 비범위

- Vision LLM 직접 입력 전환
- Qwen 메인 추출기 전환
- 신규 학습 데이터셋 구축
- OCR 백본 변경
