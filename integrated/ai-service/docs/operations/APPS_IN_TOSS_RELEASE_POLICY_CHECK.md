# Apps in Toss Release Policy Check

> 기준일: 2026-04-29  
> 대상 서비스: 물무물무 식재료 관리 서비스  
> 판단 방식: Apps in Toss 공식 문서 근거 기반. 추측성 판단은 제외한다.

## 1. 결론

물무물무의 1차 출시 범위가 아래 기능이라면 Apps in Toss 출시 가능성이 있다.

- 영수증 OCR 기반 식재료 추출
- 식재료 등록 및 목록 관리
- 소비기한 계산 및 알림
- 보유 식재료 기반 레시피 추천

다만 아래 기능은 출시 전 운영 정책과 안전 장치가 필요하다.

- 이웃 나눔 게시글
- 사용자 간 채팅
- 사용자 생성 콘텐츠
- AI 결과 노출

현재 정책 문서에서 “식재료 관리”, “영수증 OCR”, “소비기한 관리”, “레시피 추천” 자체가 출시 불가 카테고리로 명시되어 있지는 않다. 반면, 채팅과 사용자 간 연결 기능은 별도 안전 요건이 명시되어 있으므로 구현과 운영 정책 없이 출시하면 반려 또는 운영 제한 리스크가 있다.

## 2. 공식 근거

### 2-1. 출시 불가 서비스 기준

Apps in Toss 서비스 오픈 정책은 아래 유형을 출시 불가 또는 제한 대상으로 본다.

- 디지털 자산 및 가상자산 관련 서비스
- 자금세탁 가능성이 있는 현금 또는 유사 자산 교환/전환/환불 구조
- 불법 또는 부정행위 조장 서비스
- 사행성 및 복권/베팅성 콘텐츠
- 금융 상품 중개, 판매, 광고
- 투자 자문, 리딩방, 유료 정보 제공
- 의료 관련 직접 연결, 비대면 진료, 병원 예약, 병원 광고 수익 구조
- 내부 정책상 승인 불가 서비스

물무물무의 식재료 관리, OCR, 소비기한, 레시피 추천은 위 금지 유형에 직접 해당하지 않는다.

근거: [서비스 오픈 정책](https://developers-apps-in-toss.toss.im/intro/guide.md)

### 2-2. 쇼핑몰/결제 관련 조건

서비스 오픈 정책은 쇼핑몰의 경우 고객센터, 가품 방지, 환불 정책이 없으면 출시할 수 없다고 명시한다. 또한 실물 상품 결제는 토스페이만 사용할 수 있고, 외부 결제창이나 외부 가입/결제 흐름은 제한된다.

따라서 물무물무의 이웃 나눔이 “무료 나눔”이면 쇼핑몰/결제 리스크는 낮다. 반대로 돈을 받고 판매하거나 외부 계좌이체, 외부 결제, 외부 거래 링크를 유도하면 정책 리스크가 커진다.

근거: [서비스 오픈 정책](https://developers-apps-in-toss.toss.im/intro/guide.md)

### 2-3. 외부 링크 제한

Apps in Toss는 자사 앱 설치 유도와 외부 링크 이동을 제한한다. 주요 기능이 외부 링크에 의존하거나, 외부 결제창, 앱 다운로드, 외부 가입으로 이동하는 구조는 제한 대상이다.

따라서 물무물무는 OCR, 식재료 관리, 레시피 추천, 나눔 신청, 신고 등 핵심 흐름을 미니앱 안에서 완결해야 한다. 외부 앱 설치, 외부 웹 가입, 외부 결제 페이지 이동을 핵심 흐름으로 두면 안 된다.

근거: [서비스 오픈 정책](https://developers-apps-in-toss.toss.im/intro/guide.md)

### 2-4. 생성형 AI 고지 의무

공식 정책은 생성형 AI를 활용해 텍스트, 이미지, 음성, 영상 등 결과물을 생성하거나 자동 응답, 요약, 추천, 생성 기능을 사용자에게 노출하는 경우 아래를 요구한다.

- 사용자가 처음 이용하거나 AI 기능을 최초로 사용하는 시점에 AI 활용 사실을 고지
- AI가 생성한 결과물임을 사용자가 인식할 수 있도록 표시

물무물무의 레시피 추천과 소비기한 계산은 AI 또는 AI 보조 결과를 사용자에게 노출하는 기능이다. 따라서 “AI 추천 결과”, “예측 결과로 실제 보관 환경에 따라 달라질 수 있음” 같은 라벨과 안내가 필요하다.

근거: [서비스 오픈 정책](https://developers-apps-in-toss.toss.im/intro/guide.md)

### 2-5. 채팅 서비스 안전 요건

공식 채팅 서비스 주의사항은 1:1 메시지 교환 기능을 제공하는 경우 아래 기능을 요구한다.

- 신고 기능
- 차단 기능
- 반복 위반자 계정 제한 또는 정지 정책
- 신고 접수 후 운영자 확인 및 조치 프로세스
- 성매매, 사기, 불법 광고, 스팸, 불법 정보 유통 방지
- 메시지와 개인정보 보호
- 탈퇴 시 개인정보 파기
- 법적 요청 대응 프로세스
- 출시 전 이용자 보호 조치, 운영 약관, 불법 행위 대응 프로세스 제출

물무물무의 이웃 나눔 채팅은 채팅 서비스 주의사항에 해당할 수 있다. 따라서 채팅을 1차 출시 범위에 넣으려면 신고/차단/운영자 처리 정책이 먼저 구현되어야 한다.

근거: [채팅 서비스 주의사항](https://developers-apps-in-toss.toss.im/intro/_caution-chat.md)

### 2-6. 만남/소셜 서비스 오분류 방지

공식 문서는 랜덤 매칭, 근처 사용자 탐색, 공개 프로필 기반 메시지 발송, 이성 추천 또는 만남 목적 수익 모델이 있으면 채팅이 아니라 만남/소개팅 서비스로 분류될 수 있다고 설명한다.

물무물무의 나눔 기능은 식재료 나눔 목적이어야 하며, 사람 탐색이나 만남 유도 기능처럼 보이지 않아야 한다. 사용자 프로필 중심 추천, 근처 사람 탐색, 랜덤 채팅, 공개 프로필 메시지 발송은 피하는 것이 안전하다.

근거: [채팅 서비스 주의사항](https://developers-apps-in-toss.toss.im/intro/_caution-chat.md), [만남/소개팅 주의사항](https://developers-apps-in-toss.toss.im/intro/_caution-social.md)

### 2-7. 출시 절차와 기술 점검

Apps in Toss 출시 문서는 검토 요청 전 체크리스트 확인, 테스트 1회 이상 완료, 번들 업로드, 검토 요청, 승인 후 출시 흐름을 안내한다. 검토는 영업일 기준 최대 3일이 걸릴 수 있다.

기술적으로는 아래 조건을 확인해야 한다.

- 앱 번들은 압축 해제 기준 100MB 이하
- 실제 서비스 환경 Origin: `https://<appName>.apps.tossmini.com`
- QR 테스트 환경 Origin: `https://<appName>.private-apps.tossmini.com`
- CORS, 네트워크 요청, 권한 처리, 로그인/세션 유지 실제 환경 재검증
- 출시 후 오류 로그, API 응답 지연/실패율, 사용자 피드백 모니터링

근거: [미니앱 출시](https://developers-apps-in-toss.toss.im/development/deploy.md), [서비스 오픈 프로세스](https://developers-apps-in-toss.toss.im/intro/onboarding-process.md)

## 3. 물무물무 기능별 출시 판단

| 기능 | 판단 | 근거 | 출시 전 필요 조치 |
|---|---|---|---|
| 영수증 OCR | 가능성 높음 | 금지 서비스 유형에 직접 해당 없음 | 이미지 권한, OCR 실패/수동수정 UX, 개인정보 최소 수집 |
| 식재료 등록/관리 | 가능성 높음 | 생활 관리 기능으로 볼 수 있음 | 사용자 데이터 삭제/탈퇴 정책 |
| 소비기한 계산 | 가능성 높음 | 의료 기능이 아니라 식재료 보관 보조 정보 | 예측/참고 정보 고지, 책임 제한 문구 |
| 레시피 추천 | 가능성 높음 | AI 추천 결과 노출 가능, 단 AI 고지 필요 | AI 사용 고지, 추천 결과 라벨 |
| 이웃 나눔 게시글 | 조건부 가능 | 금전 거래가 없으면 쇼핑몰/결제 리스크 낮음 | 무료 나눔 명시, 금전 거래 금지, 신고/관리자 처리 |
| 채팅 | 조건부 가능 | 채팅 서비스 주의사항 적용 | 신고, 차단, 계정 제재, 메시지 보호, 운영 약관 |
| 유료 판매/거래 | 위험 | 실물 상품 결제는 토스페이만 허용, 외부 결제 제한 | 1차 출시 범위에서 제외 권장 |
| 외부 앱/웹 유도 | 위험 | 자사 앱 설치/외부 링크 제한 | 핵심 기능은 미니앱 내 완결 |

## 4. 1차 출시 권장 범위

1차 출시에서는 아래 기능만 포함하는 것이 정책 리스크가 가장 낮다.

- 토스 로그인 기반 사용자 식별
- 영수증 촬영 또는 업로드
- OCR 분석 결과 확인 및 수정
- 식재료 등록
- 소비기한 계산
- 내 식재료 목록
- 레시피 추천

이 범위는 사용자 간 직접 연결, 금전 거래, 채팅, 사용자 생성 게시글 리스크를 최소화한다.

## 5. 2차 출시 또는 후속 업데이트 권장 범위

이웃 나눔과 채팅은 아래 조건을 만족한 뒤 후속 업데이트로 넣는 것이 안전하다.

- 무료 나눔만 허용
- 금전 거래, 계좌번호, 외부 결제 유도 금지
- 식품 안전 안내 고지
- 게시글 신고 기능
- 사용자 차단 기능
- 관리자 신고 처리
- 반복 위반자 제재 정책
- 채팅 메시지 저장/보호 정책
- 탈퇴 시 개인정보 삭제 정책
- 운영 약관과 불법 행위 대응 프로세스 준비

## 6. 앱 소개 문구 방향

검수 관점에서 안전한 포지셔닝:

- “영수증으로 식재료를 등록하고 소비기한과 레시피를 관리하는 서비스”
- “냉장고 속 식재료를 더 잘 쓰도록 돕는 생활 관리 미니앱”
- “보유 식재료 기반 레시피 추천과 소비기한 관리”

피해야 할 포지셔닝:

- “동네 식재료 거래”
- “주변 사람과 식품 사고팔기”
- “근처 사용자와 자유롭게 만나는 나눔 서비스”
- “외부 앱에서 더 많은 기능 제공”

## 7. 최종 판단

물무물무는 “식재료 관리 + OCR + 소비기한 + 레시피 추천” 중심으로는 출시 가능성이 있다. 다만 “이웃 나눔 + 채팅”을 포함하면 채팅 서비스 안전 요건과 사용자 생성 콘텐츠 관리 책임이 발생한다.

따라서 현재 프로젝트 방향은 아래처럼 정리한다.

- 1차 출시: 식재료 관리/소비기한/레시피 추천 중심
- 2차 출시: 무료 나눔과 채팅은 신고/차단/운영 정책 구현 후 추가
- 금지: 현금 거래, 외부 결제, 외부 앱 설치 유도, 만남/랜덤 매칭처럼 보이는 기능

이 판단은 Apps in Toss 공식 문서 기준이며, 최종 승인은 Apps in Toss 검수 결과에 따른다.
