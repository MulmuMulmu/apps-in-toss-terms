# Normal Input Criteria

## 목적

이 문서는 현재 OCR 파이프라인이 정상 입력으로 간주하는 영수증 이미지 조건을 정의한다.

핵심 원칙:

- 모든 영수증과 모든 촬영 품질을 지원 대상으로 보지 않는다.
- 최근 스마트폰으로 일반적으로 촬영한 영수증을 주 입력 분포로 둔다.
- 이 기준을 벗어나면 파서 고도화가 아니라 재촬영 유도로 처리한다.

## 정상 입력 정의

현재 시스템은 아래 조건을 만족하는 이미지를 정상 입력으로 본다.

### 1. 이미지 품질

- 글자가 육안으로 읽힌다.
- 전체적으로 초점이 맞아 있다.
- 심한 모션 블러가 없다.
- 지나친 노이즈나 압축 깨짐이 없다.

### 2. 프레이밍

- 영수증 상단과 하단이 프레임 안에 포함된다.
- 좌우 가장자리 일부가 잘리지 않는다.
- 영수증 본문이 이미지의 주 피사체다.
- 영수증이 지나치게 멀리 찍히지 않았다.

### 3. 기하 구조

- 심한 원근 왜곡이 없다.
- 회전이 있더라도 사람이 보기에도 명확히 영수증 텍스트가 읽히는 수준이다.
- 영수증이 크게 접히거나 휘어 있지 않다.

### 4. 조명

- 강한 반사광이 텍스트 위를 덮지 않는다.
- 그림자가 품목 구간을 가리지 않는다.
- 배경과 영수증의 명암 차이가 어느 정도 확보된다.

### 5. 영수증 내용 구조

- 편의점, 마트, 일반 POS 영수증과 유사한 구조를 가진다.
- 날짜, 품목, 합계가 실제 영수증 내부에 존재한다.
- 품목 구간이 안내문/이벤트 문구에 의해 완전히 덮이지 않는다.

## 현재 우선 지원 입력

현재 구현과 테스트 기준으로 우선 지원 대상으로 보는 유형은 아래다.

- 편의점 영수증
- 마트 / 슈퍼 영수증
- 일반 POS 계산 영수증
- 바코드 + 수량/금액 상세형 영수증

## 제한 지원 입력

아래 유형은 동작할 수는 있지만 정확도 보장이 약하다.

- 카페 영수증
- 베이커리 영수증
- 배달 / 포장 영수증
- 키오스크 영수증
- 비식품이 많이 섞인 영수증

## 비정상 입력 예시

아래는 현재 기준으로 비정상 입력 또는 재촬영 권장 입력이다.

- 상단 날짜 영역이 잘린 이미지
- 하단 합계 영역이 잘린 이미지
- 상품명 글자가 번져 육안 판독이 어려운 이미지
- 손가락이 영수증 일부를 가리는 이미지
- 강한 반사광으로 본문이 날아간 이미지
- 영수증이 너무 멀리 찍혀 글자 높이가 매우 작은 이미지
- 바닥/책상 패턴이 더 두드러져 영수증 경계가 약한 이미지

## 제품 관점의 해석

현재 시스템 목표는 "사용자가 정상적으로 찍어 올린 영수증에서 품목, 수량, 날짜를 구조화한다"이다.

따라서 다음 두 가지를 동시에 유지해야 한다.

1. 정상 입력에서는 자동 추출 성공률을 끌어올린다.
2. 비정상 입력에서는 억지로 추출하지 말고 재촬영을 유도한다.
